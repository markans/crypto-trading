Bybit Signal Workflow
=====================

Purpose
-------

Use this workflow when you want to analyze a Bybit symbol and produce a trading
signal with commentary. This repo is signal-only. Nothing in it places, opens,
or submits orders. Acting on a signal is always a manual decision made outside
of these scripts.

The flow is:

1. Read public Bybit (or fallback) market data.
2. Score the trend and report recommendation, confidence, levels, and the
   risk/reward of a suggested plan.
3. Treat all output as commentary. It does not approve, reject, or place a
   trade.


Core Skill
----------

Use the installed skill:

  generating-trading-signals

Signal analysis should follow the core generating-trading-signals skill. This
file only defines the local Bybit data path, symbol format, and signal flow for
this repo.

Use this workflow for prompts such as:

  get trading signals
  check indicators
  analyze signals

For this repo, use the skill together with these local scripts:

  scripts/bybit_analyze_symbol.py   read-only analyzer
  scripts/bybit_signal.py           analyzer + suggested plan + log + email
  scripts/bybit_signal_plan.py      suggested entry/SL/TP plan for a chosen side

All of these are read-only. None of them place orders.


Confirmed Bybit Data Access
---------------------------

The analyzer reaches the default Bybit demo public market-data endpoint:

  python3 scripts/bybit_analyze_symbol.py --symbol BTCUSDT
  -> https://api-demo.bybit.com

Live public market data:

  env BYBIT_BASE_URL=https://api.bybit.com python3 scripts/bybit_analyze_symbol.py --symbol BTCUSDT
  -> https://api.bybit.com

These are unsigned public endpoints and need no API credentials. The repo no
longer uses signed (trading) endpoints at all.


Important Repo Rules
--------------------

Use Bybit symbol format without a slash (the scripts normalize automatically):

  BTC/USDT -> BTCUSDT
  ETH/USDT -> ETHUSDT
  SOL/USDT -> SOLUSDT

Do not commit .env.

For signal generation, .env only needs a market-data source and (optionally)
email settings:

  BYBIT_BASE_URL=https://api-demo.bybit.com
  SMTP_HOST=smtp.gmail.com
  SMTP_PORT=587
  SMTP_USERNAME=you@example.com
  SMTP_PASSWORD=your_app_password
  SMTP_USE_TLS=true
  EMAIL_FROM=you@example.com
  EMAIL_TO=you@example.com


Read-Only Signal Command
------------------------

Run the analyzer:

  python3 scripts/bybit_analyze_symbol.py --symbol BTCUSDT

Examples:

  python3 scripts/bybit_analyze_symbol.py --symbol ETHUSDT
  env BYBIT_BASE_URL=https://api.bybit.com python3 scripts/bybit_analyze_symbol.py --symbol BTCUSDT

The analyzer prints:

  symbol
  last_price
  24h_change_pct
  15m_local_high
  15m_local_low
  total_score
  recommendation
  confidence
  15m / 60m / 240m score, EMA21, EMA55, RSI14

Interpretation for commentary only:

  confidence=strong or moderate -> stronger candidate
  confidence=low                -> weaker signal / higher caution
  recommendation=Buy            -> analyzer leans long
  recommendation=Sell           -> analyzer leans short

Prefer setups where 60m and 240m agree with the recommendation.


Signal Generator (with plan + log + email)
-------------------------------------------

  python3 scripts/bybit_signal.py --symbol BTCUSDT --margin 25 --leverage 5 --tp-sl-mode structure --structure-interval 60

This:

  - prints the signal
  - builds a suggested entry/SL/TP plan when confidence is strong
  - appends a record to SIGNAL-LOG.md
  - emails the signal when SMTP is configured

Useful flags:

  --no-email                  do not send any email
  --email-min-confidence X    only email at/above low|moderate|strong (default strong)
  --no-log                    do not write to SIGNAL-LOG.md
  --ny-time HH:MM             only run inside a New York time window

margin and leverage size the suggested plan only. No funds are committed.


Suggested Plan Only
-------------------

When you only want a plan for a chosen side (no scoring, no email):

  python3 scripts/bybit_signal_plan.py --symbol BTCUSDT --side Buy --margin 25 --leverage 5 --tp-sl-mode structure --structure-interval 60
  python3 scripts/bybit_signal_plan.py --symbol BTCUSDT --side Sell --margin 25 --leverage 5 --tp-sl-mode structure --structure-interval 240

It prints:

  symbol, side, margin_usdt, leverage, notional_usdt, entry_reference, qty,
  stop_loss, take_profit, target_source

This is a suggestion only. It never places an order.


Risk/Reward Check
-----------------

For a Buy:

  risk   = entry_reference - stop_loss
  reward = take_profit - entry_reference

For a Sell:

  risk   = stop_loss - entry_reference
  reward = entry_reference - take_profit

Reward/risk = reward / risk. Comment if it is below 1.5.

Also comment if:

  stop_loss or take_profit is too close to entry
  stop_loss or take_profit is on the wrong side of entry
  qty is larger than intended
  leverage is higher than intended

For a Buy:  stop_loss < entry_reference < take_profit
For a Sell: take_profit < entry_reference < stop_loss


Logging Template
----------------

Each signal record contains:

  date_time
  symbol
  analyzer_recommendation
  confidence
  15m_score / 60m_score / 240m_score
  side_considered
  entry_reference
  stop_loss
  take_profit
  reward_risk
  margin_usdt
  leverage
  qty
  signal_comment
  reason
  result_r_multiple


Safety Notes
------------

This is not financial advice.

These scripts only read public market data. They do not place, open, modify, or
cancel any orders or positions.

The signal can be wrong. Treat it as commentary. Any trade you make from it is
your own manual decision.

Never share SMTP or API secrets in prompts or commit them to git.
